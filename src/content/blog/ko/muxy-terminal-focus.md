# Muxy — 여러 프로젝트를 동시에 처리할 때 집중력을 지켜주는 터미널

4~8개의 프로젝트를 동시에 처리하려고 해본 적이 있다면, 이 고통을 잘 알 것입니다. 저도 예전에는 모든 것을 한눈에 보려고 데스크톱에 창들을 이리저리 흩어놓곤 했습니다 — 솔직히 말해서 익숙해지는 데 시간이 걸리는 방법이죠.

그러다 [Muxy](https://github.com/muxy-app/muxy)를 발견했습니다. 모든 vibecoder가 지금 당장 사용해봐야 할 터미널입니다. 이 도구는 멀티태스킹 중 집중력을 확실히 높여줬습니다 — 작업이 눈에 띄게 편안해졌습니다.

한 가지 주의할 점: macOS가 다운로드한 빌드를 보안 위협으로 표시합니다. 하지만 우리는 엔지니어입니다 — 소스 코드가 완전히 공개되어 있습니다. LLM으로 보안 감사를 돌리고 Xcode에서 직접 빌드하면 됩니다.

아이러니하게도, 저는 이전에 vibecoding으로 비슷한 도구를 직접 개발하려 했습니다 — [Open Agent Manager](https://open-agent-manager.suenot.com/). 하지만 실제로 웹 기술로 터미널을 만드는 건 그다지 좋은 아이디어가 아니었습니다. 기대만큼 잘 작동하지 않습니다. 이런 종류의 도구에는 네이티브 구현이 필요하고(다행히 이제 모든 언어에서 가능합니다), 처음부터 그렇게 했어야 합니다.

최근 Warp가 오픈소스가 된 것을 보고, 제 작품을 Warp 기반으로 포팅하기 시작했습니다. 언젠가는 Warp 기반 솔루션을 완성할지도 모르겠습니다. 하지만 지금은 — Muxy를 꼭 한번 사용해보시길 강력히 추천합니다. 정말 도움이 됩니다.
